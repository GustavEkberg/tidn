## Codebase Patterns

- **AppLayer in `lib/layers.ts`**: `Layer.mergeAll(Auth.Live, Db.Live, Email.Live, S3.Live)` — all services merged here
- **Service removal**: delete dir + remove import/reference from layers.ts. No other cross-references in this codebase for Telegram/Activity.
- **Schema cleanup**: when removing a table, also remove it from `defineRelations()` table map and all relation definitions referencing it.
- **No test files remain** after post removal — all tests were scaffolding in `lib/core/post/`.
- **Adding a table**: define table with `pgTable()`, export `Type` and `InsertType`, add to `defineRelations()` table map + define relation entries for both sides.
- **Server action template**: `'use server'` → input schema with `S.Struct` → `S.decodeUnknown` with `mapError` to `ValidationError` → `getSession()` → `yield* Db` → business logic → `.pipe(Effect.withSpan, Effect.provide(AppLayer), Effect.scoped, Effect.matchEffect({onFailure: Match on _tag, onSuccess: revalidate + return}))`
- **Domain query template**: `getSession()` inside query → `yield* Db` → query logic → `.pipe(Effect.withSpan('Domain.queryName'))` — no `AppLayer` or `matchEffect` (caller provides layer + error handling)
- **Access guard pattern**: `getTimelineAccess(timelineId, requiredRole)` returns `{ timeline, role }`. Use in actions instead of inline ownership checks. Role hierarchy: owner(3) > editor(2) > viewer(1). Non-members get `NotFoundError` (no info leak). Import `TimelineRole` from `get-timelines.ts`.
- **ConstraintError**: `lib/core/errors/index.ts` — use for duplicate/constraint violations (e.g., duplicate invite). Has `message` + `constraint` fields.
- **Email from action**: use `Config.string('APP_NAME')` and `Config.string('EMAIL_SENDER')` directly in action gen — AppLayer satisfies these. `yield* Email` for the service, then `email.sendEmail({...})`.
- **better-auth databaseHooks**: add `databaseHooks` option to `betterAuth()` config. `user.create.after` fires after user creation. The `authDb` Drizzle instance from the Effect.gen closure is accessible in hook callbacks. Always wrap in try/catch — hooks must not fail user creation.
- **Cursor-based pagination**: use `(date, id)` pair for keyset pagination on non-unique columns. Fetch `limit + 1` to detect next page. Cursor condition: `OR(col < cursor, AND(col = cursor, id < cursor.id))` for desc order, flip `<` to `>` for asc. Return `{ items, nextCursor }` where nextCursor is null when no more pages.
- **Set-based mime/enum validation**: use `ReadonlySet<string>` with `.has()` instead of `ReadonlyArray<string>` with `.includes()` — avoids `as` casts that ESLint forbids. Declare sets from const tuples: `const SET: ReadonlySet<string> = new Set(TUPLE)`.
- **Media action event→timeline lookup**: media actions receive `eventId`, must fetch event first to get `timelineId`, then call `getTimelineAccess(timelineId, role)`. This two-hop pattern is shared across all media actions.
- **Fire-and-forget async processing**: use `effect.pipe(Effect.provide(AppLayer), Effect.scoped, Effect.tapError(...), Effect.catchAll(() => Effect.void), Effect.forkDaemon)` — provide layer before forking so fiber is self-contained; daemon fibers outlive parent scope.
- **Sharp photo processing**: `sharp(buffer).metadata()` for EXIF extraction, `.rotate().withMetadata({ orientation: undefined }).toBuffer()` to strip metadata + auto-rotate, `.resize({ width: N, withoutEnlargement: true }).jpeg({ quality: 80, mozjpeg: true })` for thumbnails. Use original buffer for thumbnail (pre-strip) to preserve quality. Get final dimensions from stripped buffer (post-rotation may swap width/height).
- **Thumbnail key convention**: `{originalKey}-thumb.{ext}` — e.g., `timelines/x/y/file.jpg` → `timelines/x/y/file-thumb.jpg`. Thumbnails are always JPEG regardless of original format.
- **Process-media error recovery**: `Effect.tapError` on the outer pipeline to set `processingStatus='failed'` in DB. The DB update inside tapError is wrapped in its own `catchAll` to prevent double-failure cascades.
- **Video processing via ffmpeg**: `ffmpeg-static` + `ffprobe-static` provide binary paths (no system dep). Use `execFile` (not `spawn`) wrapped in `Effect.async`. Temp dir via `Effect.acquireUseRelease` for guaranteed cleanup. `ffprobe -print_format json -show_streams -show_format` for metadata, `ffmpeg -ss N -i input -frames:v 1` for frame extraction. Parse ffprobe JSON with runtime type guards (no `as` casts). Video thumbnails always `.jpg` extension regardless of source format.
- **pnpm onlyBuiltDependencies**: packages requiring postinstall scripts (like `ffmpeg-static`, `ffprobe-static`) need to be listed in `package.json` under `pnpm.onlyBuiltDependencies` for pnpm to run their build scripts.
- **HEIC→JPEG conversion**: Sharp has libheif built-in — no extra deps. Detect via `ReadonlySet<string>` of mime types (`image/heic`, `image/heif`). Append `.jpeg({ quality: 90 })` to sharp pipeline. Replace S3 key extension (`.heic` → `.jpg`), delete old key, update media record's `s3Key` + `mimeType`. Thumbnails always `.jpg` regardless of source.
- **Page → client component pattern**: Server page (Suspense+Content) fetches data via Effect pipeline, passes plain serializable data as props to `'use client'` component. Client component handles interactivity (dialogs, forms, transitions). Define local types in client component matching the serialized shape — don't import server types. Use `useTransition` for server action calls.
